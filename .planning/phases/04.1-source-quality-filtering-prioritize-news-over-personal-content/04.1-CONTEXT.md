**Gathered:** 2026-03-25

## Phase Boundary

Curate higher-quality RSS sources, enhance the enrichment pipeline with content type classification and thinContent persistence, improve both enrichment and chat system prompts using a UX research skill, and evaluate enrichment models. Includes the folded todo "Evaluate enrichment model and prompt for better briefing quality."

<decisions>
## Implementation Decisions

### Source curation
- **D-01:** Drop Dev.to entirely. Too much noise (personal posts, beginner tutorials) for an AI news tracker.
- **D-02:** Keep both design sources: TLDR Design and Smashing Magazine. Design is relevant to the user's frontend developer profile.
- **D-03:** Research and add new high-quality AI/ML news feeds. Current mix is light on dedicated AI coverage beyond TechCrunch AI and TLDR AI. Researcher should identify candidates.
- **D-04:** Source changes are implemented in `prisma/seed.ts` (remove Dev.to entry, add new feeds via upsert pattern).

### Content type taxonomy
- **D-05:** Add `contentType` as a single-value String field on Article model. 5 types: `news`, `tutorial`, `opinion`, `announcement`, `roundup`.
- **D-06:** One contentType per article (not an array). An article is primarily one thing.
- **D-07:** Store `thinContent` as a Boolean field on Article model. Currently computed in enrichment but discarded; promote to a stored column.
- **D-08:** Both fields added in a single Prisma migration alongside any seed.ts source changes.

### Prompt improvement
- **D-09:** Apply the `chatbot-prompt-design` skill to improve both enrichment and chat system prompts. This is a separate step before model evaluation.
- **D-10:** Enrichment prompt improvements target: summary quality (more insightful, less generic), importance scoring accuracy (better calibration), and topic classification precision. Plus new contentType classification.
- **D-11:** Chat prompt improvements target: conversational quality based on UX research findings from the skill.
- **D-12:** Prompt improvement happens during planning: the researcher reads the skill and the planner drafts improved prompts as a deliverable for the executor.

### Model evaluation
- **D-13:** Structured model evaluation after prompt improvement. Run improved prompts through 2-3 candidate models on the same test batch. Compare with defined criteria (summary quality, scoring accuracy, topic precision, contentType accuracy).
- **D-14:** Cost-conscious: small test batch, budget-aware model candidates. Evaluate after Dev.to articles are removed (smaller corpus = cheaper evaluation).
- **D-15:** Model candidates from the pending todo: `deepseek/deepseek-v3.2`, `google/gemini-3.1-flash-lite-preview`, `openai/gpt-4.1-mini`, current `google/gemini-2.5-flash-lite` as baseline.

### Backfill strategy
- **D-16:** Delete all Dev.to articles from the database (source removal cleanup).
- **D-17:** Full re-enrichment of remaining articles using the improved prompt and chosen model. Not a classification-only pass; everything gets fresh summaries, scores, topics, contentType, and thinContent.
- **D-18:** One-time Node script for the re-enrichment (standalone, like the Phase 03.2 data migration). Explicit control over batch size and progress.

### No filtering changes
- **D-19:** Briefing page stays as-is. Once sources are better quality, everything is eligible. No content type filtering or demotion in the briefing.
- **D-20:** Chat tools stay unfiltered. No contentType filtering in search, topic, or recent article tools.
- **D-21:** contentType and thinContent are stored for future use (Phase 5 UX polish or later), not actively used for filtering in this phase.

### Folded todos
- **D-22:** "Evaluate enrichment model and prompt for better briefing quality" (area: ai-enrichment) folded into this phase. Covered by D-09 through D-15.

### Claude's Discretion
- Exact new RSS feed URLs (researcher proposes, planner finalizes)
- Migration script batch size and progress reporting
- Schema details (column defaults, nullability for existing articles)
- Enrichment prompt exact wording (guided by chatbot-prompt-design skill output)
- Model evaluation scoring methodology and comparison format
- Order of operations within execution plans

</decisions>

<specifics>
## Specific Ideas

- Source quality is the primary lever. Better sources mean everything they produce is briefing-worthy, no filtering needed.
- The chatbot-prompt-design skill is based on UX research into how real users interact with chatbots. Apply its findings to both the enrichment analyst prompt and the chat assistant prompt.
- Model evaluation should happen after prompt improvement, so the improved prompts are the ones being compared across models.
- Dev.to articles should be deleted from the DB, not just orphaned by removing the source.

</specifics>
