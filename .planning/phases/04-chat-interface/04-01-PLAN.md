       ```prisma

         count       Int      @default(0)
         windowStart DateTime @default(now())

       }
       ```
    2. Run `npx prisma migrate dev --name add-rate-limit` to create migration.
    3. Create src/lib/rate-limit.ts with:
       - `import 'server-only'` at top
       - `import { prisma } from './prisma'`
       - Constants: `WINDOW_MS = 60 * 60 * 1000` (1 hour), `MAX_MESSAGES = 20` (per D-05)
       - `export async function checkRateLimit(key: string): Promise<number | null>` -- finds rate limit record where `key` matches AND `windowStart >= new Date(Date.now() - WINDOW_MS)`. If no record or `record.count < MAX_MESSAGES`, return null. Otherwise return `Math.ceil((record.windowStart.getTime() + WINDOW_MS - Date.now()) / 60000)` (minutes until reset).
       - `export async function incrementRateLimit(key: string): Promise<void>` -- finds existing record for key with valid window. If exists, `prisma.rateLimit.update` incrementing count by 1. If no valid-window record exists, `prisma.rateLimit.upsert` with key, reset count to 1 and windowStart to `new Date()`. Use `upsert` where `{ key }` with create `{ key, count: 1, windowStart: new Date() }` and update `{ count: 1, windowStart: new Date() }` for the expired-window case.
    4. Create src/lib/rate-limit.test.ts following established patterns:
       - `vi.mock('server-only', () => ({}))`
       - `vi.mock('@/lib/prisma', () => ({ prisma: { rateLimit: { findFirst: vi.fn(), update: vi.fn(), upsert: vi.fn() } } }))`
       - Import functions, write tests per behavior spec above
       - Use `vi.spyOn(Date, 'now')` or manual date math for time-window assertions
