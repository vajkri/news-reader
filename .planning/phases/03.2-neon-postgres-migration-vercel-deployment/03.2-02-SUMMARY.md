## Decisions Made

- **better-sqlite3 for SQLite reads**: Using direct better-sqlite3 SQL reads (not Prisma) avoids the Prisma dual-engine conflict. After schema switches to Postgres, the generated Prisma client binary cannot open SQLite files.
- **Numeric timestamp comparison**: SQLite stores `publishedAt` as Unix millisecond integers. The WHERE clause uses `CUTOFF.getTime()` (numeric), not `.toISOString()` (string) for correct comparison.
- **Auto-advance Task 2**: `auto_advance: true` in config.json; data verified programmatically (6 sources, 236 articles, enrichment sample confirmed correct) before auto-approving.
