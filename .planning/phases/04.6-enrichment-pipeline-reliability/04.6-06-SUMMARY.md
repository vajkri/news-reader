## What Was Built

### buildCalibrationContext (src/lib/enrich.ts)

- Queries `articleFeedback` for last 30 days via `prisma.articleFeedback.findMany`
- Returns empty string when no feedback or all counts below threshold (no noise injection)
- Source-level signals: 3+ same-direction votes from one source produce a calibration line
- Reason-level signals: 5+ votes citing one reason produce a calibration line with count
- Returns `\n\n## User Calibration Signals\n...` section appended to SYSTEM_PROMPT

### enrichArticlesBatch integration

- Calls `buildCalibrationContext()` before `generateText`
- Appends result to `SYSTEM_PROMPT + calibration` as the `system` parameter
- Empty calibration produces no change to prompt behavior

### enrich.test.ts additions

- Added `articleFeedback: { findMany: vi.fn() }` to prisma mock
- Added `mockFeedbackFindMany` typed mock ref
- Added `buildCalibrationContext` to imports
- Fixed `enrichArticlesBatch` beforeEach to default-mock feedback as empty
- 5 new tests: empty feedback, below-threshold, 3+ downvotes, 3+ upvotes, 5+ reason votes
- 1 integration test: enrichArticlesBatch calls generateText with calibration in system prompt
- Total: 19 tests pass (was 13)

### Briefing page wiring (src/app/briefing/page.tsx)

- Added pending articles query: `enrichedAt: null, createdAt: { gte: pendingCutoff }` (48h window)
- Added `lastEnrichedAt` query: `findFirst` on most recently enriched article
- Renamed `pendingCount` to `devPendingCount` to avoid collision
- `StatusBar` rendered in live mode with `newCount`, `lastVisit`, `lastEnrichedAt`, `pendingCount`
- `PendingSection` rendered in live mode with serialized pending articles
- Archive mode: no StatusBar, no PendingSection (per D-05)
