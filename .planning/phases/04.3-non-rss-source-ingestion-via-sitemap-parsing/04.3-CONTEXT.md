# Phase 04.3: Non-RSS Source Ingestion via Sitemap Parsing - Context

**Gathered:** 2026-03-25
**Status:** Ready for planning

<domain>
## Phase Boundary

Add sitemap-based and HTML-scraping source ingestion alongside existing RSS, using a strategy pattern dispatch. Three fetch strategies: `rss` (existing), `sitemap` (new), and `scrape` (new). Enable Anthropic News and Claude Blog as the first non-RSS sources. All strategies return `ParsedArticle[]` so the enrichment pipeline works identically regardless of source type.

</domain>

<decisions>
## Implementation Decisions

### Content extraction
- **D-01:** Use HTML scrape + readability to extract article content from sitemap/scrape-discovered URLs. Fetch each page, extract title, description/content, and thumbnail. Gives enrichment the same quality input as RSS sources.
- **D-02:** Store extracted content in the existing `Article.description` column. No new content field. Enrichment pipeline works identically for all source types.
- **D-03:** Extract thumbnail via `og:image` (or `twitter:image`) from page meta tags. Fall back to null if missing. Existing UI handles null thumbnails.

### Sitemap filtering
- **D-04:** Each sitemap source defines a URL path pattern (e.g., `/news/*`). Only URLs matching the pattern are ingested. Simple, explicit, per-source.
- **D-05:** Apply the same 7-day age cutoff used for RSS sources. Use sitemap `lastmod` date for filtering. Consistent behavior across source types.

### Source model schema
- **D-06:** Add `sourceType String @default("rss")` to the Source model. Values: `rss`, `sitemap`, `scrape`. Fetch cron dispatches based on this field.
- **D-07:** Add `sitemapPathPattern String?` to Source model. Used by sitemap sources to filter URLs (e.g., `/news/*`). Null for RSS sources.
- **D-08:** Both fields added in a single Prisma migration alongside seed.ts source additions.

### Strategy pattern
- **D-09:** Three strategies from day one: `rss`, `sitemap`, `scrape`. User plans to add more sources soon, some without RSS or sitemaps. HTML scraping covers sites with neither.
- **D-10:** All strategies return `ParsedArticle[]` (the existing interface from `rss.ts`). This is the strategy contract.
- **D-11:** The scrape strategy fetches a listing page, extracts article links, then fetches each article page for content using the same readability approach as sitemap.
- **D-12:** Scrape source config fields are Claude's discretion; researcher should determine what's needed (CSS selector, path pattern, or both).

### New sources
- **D-13:** Add Anthropic News (`anthropic.com/news`) and Claude Blog (`claude.com/blog`) as the first non-RSS sources. Researcher to confirm whether each has a sitemap or needs scraping.

### Claude's Discretion
- Readability library choice (e.g., `@mozilla/readability` + `linkedom`, or `cheerio` with custom extraction)
- Scrape strategy config fields (CSS selector, path pattern, or both)
- How the fetch cron dispatches per sourceType (switch/case, map lookup, etc.)
- Sitemap XML parsing library choice
- Whether to add a `fetchStrategy` module or extend `rss.ts`
- Migration script details (batch size, error handling)
- `guid` generation for sitemap/scrape articles (URL-based vs content hash)

</decisions>
