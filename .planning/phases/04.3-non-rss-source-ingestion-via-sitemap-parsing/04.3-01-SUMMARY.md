## Decisions Made

- Used `vi.clearAllMocks()` instead of `vi.restoreAllMocks()` in beforeEach: `restoreAllMocks` removes the module-level vi.mock stubs, breaking mock behavior in subsequent tests. `clearAllMocks` only resets call history and return values, preserving mock stubs.
- Cast `next: { revalidate: 0 }` as `any` in sitemap.ts fetch call: TypeScript's standard `RequestInit` type does not include Next.js's extended `next` option; cast avoids a spurious type error without losing correctness.
