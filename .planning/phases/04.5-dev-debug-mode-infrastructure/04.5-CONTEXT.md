# Phase 04.5: Dev Debug Mode Infrastructure - Context

**Gathered:** 2026-03-28
**Status:** Ready for planning

<domain>
## Phase Boundary

Deliver a dev-only debug mode infrastructure with a toggle, shared state mechanism, and one concrete use case: inline debug annotations on the daily briefing page. The infrastructure must be extensible for future debug contexts (feed, chat, etc.) but this phase only implements briefing.

</domain>

<decisions>
## Implementation Decisions

### Activation Mechanism
- **D-01:** Debug mode is toggled via a floating button fixed to the bottom-right corner of the viewport
- **D-02:** The toggle and all debug UI only render when `NODE_ENV === 'development'`; zero footprint in production builds
- **D-03:** The button should be minimal and unobtrusive; it indicates current debug state (on/off)

### State Management
- **D-04:** Claude's Discretion: choose between React Context (DebugProvider + useDebug hook) or HTML data attribute pattern (like existing `data-chat-open`). Pick whichever is the best fit for a dev-only, app-wide boolean toggle that multiple components need to read.

### Debug Container Design
- **D-05:** The infrastructure supports multiple container types (inline annotations, panels, drawers) but this phase only implements inline annotations
- **D-06:** Inline annotations render directly in the page flow next to the relevant UI elements, not in a separate panel

### First Use Case: Briefing Page
- **D-07:** Between the briefing heading and the topic groups, show a debug box with: the exact timeframe (date range) for that daily briefing, and metadata about the enrich cron call that generated the data
- **D-08:** Below each BriefingCard, show the full database record for that article as nicely formatted, indented JSON (RSS data, enriched data, all stored fields)
- **D-09:** Per-card debug data is always visible when debug mode is ON; no collapse/expand per card

### Scope
- **D-10:** This phase delivers: debug toggle button, state management, briefing page debug annotations only
- **D-11:** Debug annotations for feed page, chat panel, or other contexts are future work

</decisions>

<specifics>
## Specific Ideas

- User wants "nicely formatted, indented JSON" for per-card data -- use monospace font, syntax-highlighted or at minimum indented with `JSON.stringify(data, null, 2)`
- Debug box between heading and items should show the enrichment call metadata (model used, timestamp, batch size if available)
- The infrastructure should be built so adding debug to other pages later is straightforward (just import the hook and conditionally render)

</specifics>
