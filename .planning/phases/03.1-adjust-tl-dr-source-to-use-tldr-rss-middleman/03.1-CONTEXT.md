<code_context>
## Existing Code Insights

### Reusable Assets
- `src/app/api/fetch/route.ts`: Fetch cron handler with CRON_SECRET auth, guid deduplication, Promise.allSettled batch processing
- `src/lib/rss.ts`: RSS parser with fetchFeed() function
- `src/lib/prisma.ts`: Prisma client singleton

### Established Patterns
- Source URLs are `@unique` in Prisma schema; changing a URL means updating the record or handling the constraint
- `prisma/seed.ts` uses `upsert` keyed on URL; changing the URL means the old record won't match and a new one gets created
- Guid-based deduplication in fetch route prevents re-importing existing articles

### Integration Points
- `prisma/seed.ts`: Update TLDR Tech URL, add 3 new sources (TLDR AI, TLDR Design, Hacker News)
- Database: existing Source record for "TLDR Tech" needs URL update (or new record + old cleanup)
- No API route changes needed; fetch cron automatically picks up new/updated sources

</code_context>
