# Phase 03.2: Neon Postgres Migration + Vercel Deployment - Context

**Gathered:** 2026-03-21
**Status:** Ready for planning

<domain>
## Phase Boundary

Migrate the database from local SQLite (better-sqlite3) to Neon Postgres via Vercel Marketplace. Migrate existing article data (March 18th onward) to preserve AI enrichments. Remove all SQLite dependencies. Update cron schedule to daily (Vercel Hobby limit: 2 jobs). Deploy to Vercel with working cron jobs and all pages functional.

</domain>

<decisions>
## Implementation Decisions

### Database provisioning
- **D-01:** Provision Neon via Vercel Marketplace (dashboard, manual step by user)
- **D-02:** Free tier is sufficient (0.5 GB storage, 100 CU-hours/month)
- **D-03:** Auto-select region (user is in Denmark; likely EU)
- **D-04:** No Neon branching (solo project, no staging environment needed)
- **D-05:** Standard Prisma pooled connection (no `@neondatabase/serverless` adapter needed at this scale)

### Prisma migration strategy
- **D-06:** Delete existing SQLite migrations, run fresh `prisma migrate dev --name init` against Postgres
- **D-07:** Schema changes: `provider = "postgresql"`, add `directUrl = env("DATABASE_URL_UNPOOLED")`, upgrade `topics` from `String?` to `Json?`

### Data migration
- **D-08:** Custom Node script (`scripts/migrate-data.ts`) using better-sqlite3 for SQLite read (provider-agnostic) and Prisma client for Postgres write (dual Prisma clients infeasible: query engine binary is provider-specific)
- **D-09:** Only migrate articles from March 18th 2026 00:00 onward (preserves AI enrichment tokens)
- **D-10:** Idempotent: skip existing records by `guid` (articles) and `url` (sources)
- **D-11:** Script committed to git, then deleted after migration (preserved in git history)
- **D-12:** Runs locally: reads `./prisma/dev.db`, writes to Neon via connection string from `vercel env pull`

### Dependency cleanup
- **D-13:** Remove `better-sqlite3`, `@prisma/adapter-better-sqlite3`, `@libsql/client`, `@prisma/adapter-libsql` and their `@types` packages

### Cron schedule
- **D-14:** Two daily cron jobs (Vercel Hobby plan limit): fetch at 05:00 UTC, enrich at 06:00 UTC

### Search improvement
- **D-15:** Add `mode: 'insensitive'` to Prisma `contains` filters (Postgres-only feature)
- **D-16:** Keep AND array composition pattern (it is correct, not a workaround)
- **D-17:** Full-text search deferred (seeded as SEED-001)

### Deployment
- **D-18:** Local dev points at remote Neon (no local SQLite after migration)
- **D-19:** Vercel project already linked: vajkris-projects/news-reader
- **D-20:** Guide user through Neon Marketplace integration during execution

### Claude's Discretion
- Migration script batching strategy (iterate-and-insert vs bulk)
- Prisma client initialization pattern for Neon (singleton in `src/lib/prisma.ts`)
- Whether to add a `postinstall` script for `prisma generate`
- Error handling in the migration script

</decisions>
