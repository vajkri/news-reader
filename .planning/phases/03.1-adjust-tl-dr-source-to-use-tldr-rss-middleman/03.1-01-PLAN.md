**Change 1: Add URL-swap preamble in the main() function BEFORE the upsert loop.**

Insert this block after the "Seeding sources..." log and before the `for` loop:

```typescript
// One-time fix: swap TLDR Tech URL from HN to the real TLDR feed.
// This preserves the existing source.id and all article relations.
// Safe to rerun: checks existence before updating.
const staleHnTldr = await prisma.source.findUnique({
  where: { url: "https://hnrss.org/frontpage" },
});
if (staleHnTldr?.name === "TLDR Tech") {
  await prisma.source.update({
    where: { url: "https://hnrss.org/frontpage" },
    data: { url: "https://bullrich.dev/tldr-rss/tech.rss" },
  });
  console.log("  ~ Migrated TLDR Tech URL to tldr-rss middleman");
}
```

This MUST come before the upsert loop because:
- The upsert is keyed on `url`. If the old URL is still in the DB, the new "Hacker News" source (same hnrss.org URL) would match the old TLDR Tech record instead of creating a new row.
- After the update, "TLDR Tech" has the new URL, and the upsert for "Hacker News" (hnrss.org) finds no match and creates a new record.
