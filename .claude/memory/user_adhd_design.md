User has ADHD. This shapes every design decision:
- No large bodies of text — use cards, bullet points, short blocks
- Strong visual hierarchy — clear headings, whitespace, borders
- Scannable layouts — information should be graspable at a glance
- Bite-sized content — summaries over full articles
- Consider text-to-speech compatibility (Safari on iOS handles this natively)

This is a constraint that applies to the entire product, not a feature to toggle on/off.
