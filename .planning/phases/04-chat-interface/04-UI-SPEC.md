## Spacing Scale

Multiples of 4 only. Standard 8-point scale applies across all chat panel elements.

| Token | Value | Chat Usage |
|-------|-------|------------|
| xs | 4px | Icon gaps, dot separators, inline metadata spacing |
| sm | 8px | Chip gap, panel header action gap, message bubble gap |
| md | 16px | Panel body padding, context card margin, chip padding inline, panel header vertical padding |
| lg | 24px | Panel body vertical padding (empty state), section breaks |
| xl | 32px | — |
| 2xl | 48px | — |
| 3xl | 64px | — |

Exceptions:
- Panel header padding: 16px vertical / 16px horizontal
- Chip padding: 8px vertical / 12px horizontal
- Context card left border: 4px (functional accent border, same pattern as BriefingCard `border-l-[3.5px]` — rounded to nearest multiple of 4)
- FAB (floating action button): 52px diameter — touch target minimum for mobile (exceeds 44px minimum)

## Color

Inherits the project's 60/30/10 rule. No new tokens. All values from globals.css.

| Role | CSS Token | Light | Dark | Usage |
|------|-----------|-------|------|-------|
| Dominant (60%) | `--background` / `--card` | #ffffff / #ffffff | #09090b / #0f0f12 | Page behind panel, panel shell background |
| Secondary (30%) | `--muted` / `--secondary` | #f4f4f5 | #27272a | Panel header bg tint, chip background (`#18181b` in dark), hover states |
| Accent (10%) | `--primary` | #18181b | #fafafa | FAB button fill, send button, focus ring on chat input |
| Border | `--border` | #e4e4e7 | #27272a | Panel border, chip borders, context card border, input border |
| Muted text | `--muted-foreground` | #71717a | #a1a1aa | Placeholder text, metadata, section labels, empty-state sub-copy |
| Secondary text | `--foreground-secondary` | #3f3f46 | #d4d4d8 | Chip text color (default state) |
| Destructive | N/A | — | — | No destructive actions in this phase |

Accent (`--primary`) reserved for:
1. FAB button fill (floating action button that opens the chat panel)
2. Send/submit button inside the chat input
3. Focus ring on the chat text input (`focus-visible:ring-[var(--primary)]`)

`--primary` is NOT used for: chip borders, panel background, context card border, message bubbles, or any informational surfaces.

Context card left border: `border-l-[var(--border)]` at rest — same neutral left-border pattern as BriefingCard importance tiers, but using `--border` color (not a tier color) since context cards are not importance-ranked. Source: CONTEXT.md D-19, specifics section.

Citation/source cards (below AI response): use `--card` background, `--border` border — same card pattern as BriefingCard.

## Interaction Contract

### Panel: dock and resize

- Desktop default: side panel, right edge, 350px wide (CONTEXT.md D-08)
- Mobile default: bottom panel, 25dvh height with min 200px / max 60dvh guards (CONTEXT.md D-08)
- Resize: drag handle on left edge (side mode) or top edge (bottom mode)
- Reposition: user can toggle between side and bottom via a control in panel header
- Panel does NOT cover the full viewport — current page content stays visible

### Panel: open and close

- FAB: bottom-right corner, always visible, z-index above all content
- Keyboard shortcut: `Cmd+K` (Mac) / `Ctrl+K` (Windows/Linux) — research-backed convention for quick-access overlays/chat (matches Linear, Vercel, Raycast)
- Open animation: slide in from right (side mode) or slide up from bottom (bottom mode), 200ms ease-out
- Close animation: reverse slide, 150ms ease-in
- Closing hides the panel; conversation state is preserved in React memory until page refresh (CONTEXT.md D-11)

### Generic empty state (CONTEXT.md D-14, D-15, D-16, D-17)

Layout: centered vertically in panel body
1. Icon: 48x48px rounded box, `--card` bg, `--border` border, lucide `Sparkles` icon at 22px / `--muted-foreground` color
2. Heading: "Ask about your news" — 18px / 600 / `--foreground`
3. Sub-copy: "Search, summarize, and analyze articles from your collected sources." — 14px / 400 / `--muted-foreground`
4. Prompt chips (4 hardcoded):
   - "What's new with Claude?"
   - "Top stories today"
   - "Developer tools this week"
   - "Summarize model releases"
5. Chips fade to opacity-0 with transition when user begins typing (not removed from DOM until message sent)

### Contextual empty state — "Chat about this" (CONTEXT.md D-19, D-20, D-21)

Layout: pinned context card at top of panel body, centered chips below (Option A from mockup)
1. Context card (pinned, non-scrolling):
   - Label: "Chatting about" — 13px / 600 / uppercase / `--muted-foreground`
   - Title: article title — 14px / 600 / `--foreground`, max 2 lines, line-clamp-2
   - Meta: source name + relative time — 13px / 400 / `--muted-foreground`
   - Left border: 4px solid `--border`
   - Background: `--card`, border: `--border`, border-radius: 0.625rem
   - Margin: 16px all sides inside panel body
2. Sub-heading: "What would you like to know?" — 14px / 400 / `--muted-foreground`, centered
3. Contextual chips (4 hardcoded):
   - "Summarize this article"
   - "Why does this matter?"
   - "Related articles"
   - "Compare to competitors"
4. Input placeholder changes to: "Ask anything about this article..." (CONTEXT.md D-20)
5. No auto-send — user must submit manually (CONTEXT.md D-19)

### Chat message bubbles

- User message: right-aligned, `--primary` background, `--primary-foreground` text, border-radius 1rem (rounded top-right 4px)
- Assistant message: left-aligned, `--card` background, `--border` border, `--foreground` text, border-radius 1rem (rounded top-left 4px)
- Max width: 85% of panel width
- Spacing between bubbles: 8px

### Citation / source cards (CONTEXT.md D-06)

Rendered below assistant message text when articles are cited:
- Card: `--card` background, `--border` border, border-radius 0.5rem, padding 12px
- Content: article title (14px / 600 / `--foreground`), source name as Badge (secondary variant), relative date (13px / `--muted-foreground`)
- Clickable: opens article link in new tab
- Focus: `focus-visible:ring-2 focus-visible:ring-[var(--primary)]`

### Panel header controls

- Title: "Chat" — 14px / 600 / `--foreground`
- New conversation button: icon-only Button (size `icon`, ghost variant), lucide `Plus` icon, 28x28px
- Close button: icon-only Button (size `icon`, ghost variant), lucide `X` icon, 28x28px
- Both use `focus-visible:` — never bare `focus:`

### Chat input

- `Input` component, full width inside panel footer
- Height: 40px (h-10)
- Placeholder: "Ask about any topic, source, or time period..." (generic) or "Ask anything about this article..." (contextual)
- Send button: icon-only Button (size `icon`, default variant), lucide `ArrowUp` icon
- Footer padding: 12px all sides

### Streaming

- Assistant message renders progressively as tokens stream
- A loading indicator (3 animated dots, `--muted-foreground` color) appears while waiting for first token
- Input is disabled during streaming; send button shows loading state

### Rate limit error state

- Inline error message below last message, not a modal
- Copy: "You've reached the hourly message limit. Try again in {N} minutes."
- Color: no destructive red — use `--muted-foreground` text, `--card` background card
- Input remains visible but disabled until limit resets

## Accessibility Contract

All elements must meet WCAG 2.2 AA (project rule in CLAUDE.md):

- Text contrast: 4.5:1 minimum. `--foreground` on `--card` and `--background` meets this in both modes.
- `--muted-foreground` (#71717a light, #a1a1aa dark): verify 4.5:1 against `--card` at implementation time. Use for metadata only (not primary text).
- Focus: all interactive elements use `focus-visible:ring-2 focus-visible:ring-[var(--primary)]`. Never bare `focus:`.
- FAB: `aria-label="Open chat"`, `aria-expanded` reflects panel open state
- ChatPanel: `role="complementary"` or `role="dialog"` with `aria-label="Chat"`, `aria-modal` if using dialog role
- Chat input: `aria-label="Chat message"`, `aria-disabled` when rate-limited
- Message list: `role="log"`, `aria-live="polite"` for streaming updates
- Close button: `aria-label="Close chat"`
