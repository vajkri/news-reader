## Spacing Scale

Declared values (multiples of 4 only):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, badge inner padding, inline gaps |
| sm | 8px | Card footer spacing, button gaps, tag row gaps |
| md | 16px | Card internal padding, section heading margin-bottom |
| lg | 24px | Card-to-card vertical gap, section-to-section gap |
| xl | 32px | DateStepper margin-bottom, page top padding |
| 2xl | 48px | Major section breaks between topic groups |
| 3xl | 64px | Page-level vertical breathing room (empty state centering) |

Exceptions: DateStepper row uses 8px gap between prev/label/next elements. Topic heading uses 12px bottom margin before first card (single exception, not a new token ‚Äî use `mb-3`).

Source: CLAUDE.md 8-point scale convention; exceptions derived from Claude's Discretion (card spacing).

## Color

| Role | Light mode | Dark mode | Usage |
|------|-----------|-----------|-------|
| Dominant (60%) | `--background` #ffffff | `--background` #09090b | Page background, card backgrounds |
| Secondary (30%) | `--muted` #f4f4f5 | `--muted` #27272a | Topic section headings bg, hover states |
| Accent (10%) | `--primary` #18181b | `--primary` #fafafa | Default Button (date nav), active nav link |
| Destructive | N/A | N/A | No destructive actions in this phase |

**Functional tier colors (not brand accent ‚Äî Tailwind palette tokens directly):**

| Tier | Score | Light bg | Light text | Dark bg | Dark text | Badge variant |
|------|-------|----------|------------|---------|-----------|---------------|
| Critical | 9-10 | `bg-red-100` | `text-red-800` | `bg-red-900/30` | `text-red-300` | `critical` |
| Important | 7-8 | `bg-amber-100` | `text-amber-800` | `bg-amber-900/30` | `text-amber-300` | `important` |
| Notable | 4-6 | `bg-blue-100` | `text-blue-800` | `bg-blue-900/30` | `text-blue-300` | `notable` |

Accent (`--primary`) is reserved for: default Button component, active nav link, search focus ring. NOT used for tier badges, hover states, or informational highlights.

Muted foreground (`--muted-foreground`): card footer text (source name, relative time), placeholder text, secondary labels.

Border (`--border`): card border, section dividers, DateStepper outline button border.

Source: CLAUDE.md Color section (60/30/10 rule); CONTEXT.md importance tier palette; RESEARCH.md Pattern 4 badge variant code.

### BriefingCard anatomy

```
[card border, bg-card, rounded-lg, p-4]
  [thumbnail OR topic icon fallback ‚Äî 80x80px, rounded-md]
  [right column]
    [row: Badge(tier) + Badge(secondary, topic name)]
    [headline ‚Äî text-base font-semibold, text-foreground]
    [summary ‚Äî text-sm, text-foreground, line-clamp-3 fallback]
    [footer row: source name + "," + relative time ‚Äî text-sm text-muted-foreground]
```

Card is a full-width anchor (`<a target="_blank" rel="noopener noreferrer">`). Hover state: `hover:bg-muted/50` on card background. Focus state: `focus-visible:ring-2 focus-visible:ring-primary` on anchor.

Thumbnail: `<img>` 80x80px, `object-cover rounded-md`. Fallback: lucide icon centered in 80x80 `bg-muted rounded-md` with `text-muted-foreground` icon color, size 32px.

### TopicGroup anatomy

```
[section element]
  [heading: "Topic Name (N)" ‚Äî text-xl font-semibold, text-foreground, mb-3]
  [stack of BriefingCard, gap-6]
```

No background on group ‚Äî section headings float above card stack on `--background`.

### DateStepper anatomy

```
[flex row, items-center, gap-2]
  [Button variant=outline size=icon: ChevronLeft]
  [span: date label ‚Äî text-sm font-semibold, min-w-[120px], text-center]
  [Button variant=outline size=icon: ChevronRight ‚Äî disabled when isToday]
  [Button variant=ghost size=sm: "Today" ‚Äî shown only when not today]
```

Positioned at top of briefing page, below page title, above first topic group. Margin-bottom: 32px (xl).

## Interaction Contract

| Interaction | Behavior |
|-------------|----------|
| Card click | Opens `article.url` in new tab (`target="_blank" rel="noopener noreferrer"`) |
| DateStepper prev | Navigate to `?date=YYYY-MM-DD` (previous day) via `useRouter().push()` |
| DateStepper next | Navigate to `?date=YYYY-MM-DD` (next day), disabled when `isToday(currentDate)` |
| DateStepper Today | Navigate to `/briefing` (no query param), only visible when not on today |
| Nav "Briefing" link | Navigate to `/briefing` (internal link, same tab) |
| Card hover | `hover:bg-muted/50` transition-colors on card container |
| Card focus | `focus-visible:ring-2 focus-visible:ring-offset-2 focus-visible:ring-primary` on anchor |
| Button focus | `focus-visible:` ring via existing Button component (already implemented) |

## States

| State | Trigger | Rendering |
|-------|---------|-----------|
| Loaded | Articles exist for selected date | Topic groups with cards |
| Empty | No enriched articles with importanceScore >= 4 for selected date | Empty state (see Copywriting) |
| Past date | `?date=` param is a prior day | Same layout, DateStepper shows date label + Today button |
| Today (default) | No `?date=` param | DateStepper shows "Today", next button disabled |

No loading spinner needed ‚Äî briefing page is a Server Component, ISR handles caching. No skeleton state for this phase.

## Accessibility Contract

| Concern | Implementation |
|---------|---------------|
| Contrast ‚Äî Critical badge | red-100/red-800 light, red-900/30 bg with red-300 text dark ‚Äî verify visually before ship |
| Contrast ‚Äî Important badge | amber-100/amber-800 light, amber-900/30 bg with amber-300 text dark |
| Contrast ‚Äî Notable badge | blue-100/blue-800 light, blue-900/30 bg with blue-300 text dark |
| Contrast ‚Äî body text | `--foreground` on `--background` ‚Äî pre-verified in Phase 1 |
| Contrast ‚Äî muted text | `--muted-foreground` (#71717a) on `--background` (#ffffff): 4.48:1 ‚Äî passes AA for 14px+ |
| Focus indicators | `focus-visible:` only, never bare `focus:` ‚Äî matches CLAUDE.md rule |
| Interactive cards | Wrapped in `<a>` ‚Äî receives keyboard focus naturally |
| DateStepper buttons | `aria-label` on icon-only buttons (prev/next) |
| DateStepper disabled state | `disabled` attribute on next button when isToday ‚Äî browsers suppress focus on disabled |
| Topic icons (decorative) | `aria-hidden={true}` on lucide icons in thumbnail fallback |
| Page heading | `<h1>` for "Daily Briefing", `<h2>` for topic group headings |

Target: WCAG 2.2 AA. Source: CLAUDE.md Accessibility section.
