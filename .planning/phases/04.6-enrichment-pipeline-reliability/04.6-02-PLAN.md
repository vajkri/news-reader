  <action>
Create `.github/workflows/enrich.yml` with this exact content:

```yaml
name: Fetch and Enrich

on:
  schedule:
    - cron: '0 */4 * * *'
  workflow_dispatch:

jobs:
  fetch-and-enrich:
    runs-on: ubuntu-latest
    steps:
      - name: Fetch articles
        run: |
          curl -f -s \
            -H "Authorization: Bearer ${{ secrets.CRON_SECRET }}" \
            "${{ vars.APP_URL }}/api/fetch"

      - name: Enrich articles
        run: |
          curl -f -s \
            -H "Authorization: Bearer ${{ secrets.CRON_SECRET }}" \
            "${{ vars.APP_URL }}/api/enrich"
```

Per D-06: Single workflow file, every 4 hours, calls /api/fetch then /api/enrich sequentially.
Uses `secrets.CRON_SECRET` (GitHub repo secret) and `vars.APP_URL` (GitHub repo variable for production URL).
`curl -f` ensures the step fails visibly on HTTP errors (401, 500).
`workflow_dispatch` allows manual triggering from the GitHub Actions UI.
