**2. [Rule 1 - Bug] Main re-enrich run left 56 articles with null contentType/thinContent**
- **Found during:** Task 2, post-run verification
- **Issue:** AI returned results for 15 IDs per batch but some returned IDs didn't match batch article IDs (offset pagination means earlier-enriched articles get re-sorted, causing AI to return IDs for different articles). 309 DB updates succeeded but 56 contentType/thinContent stayed null.
- **Fix:** Created `scripts/re-enrich-nulls.ts` targeting only null articles, with smaller batch (10), explicit ID validation, and warning for missing AI output IDs. All 56 fixed cleanly.
- **Files modified:** scripts/re-enrich-nulls.ts (new), scripts/check-nulls.ts (new), scripts/check-schema.ts (new)
- **Commit:** fa14e2e
