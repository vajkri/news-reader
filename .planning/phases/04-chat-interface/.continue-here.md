<decisions_made>

1. **Tool-calling over vector embeddings**: LLM calls Prisma queries as tools to search articles by keyword, date, topic, source. Zero extra cost, uses existing schema, precise results. Can add vectors later if needed.
2. **Rate limiting via Postgres table**: Prisma model in existing Neon DB. No new dependencies. Survives restarts. Migrates cleanly with Prisma.
3. **Rate limit values**: 20 messages/hour, 10-turn conversation history kept. No output token cap (control cost via message limit).
4. **Summarize button in Phase 4**: Built alongside chat since it shares same AI infrastructure (LLM call, streaming, rate limiting).
5. **Citation format: source cards below response**: Answer text first, then clickable article cards (title, source, date) underneath. Matches existing card-based design language. ADHD-friendly.
6. **Chat model: `openai/gpt-5-mini`** ($0.25/$2.00 per M tokens): GPT-5 series quality, best-in-class tool-calling, cheaper than gemini-3-flash. 400K context, more than enough.
7. **Grounding/factuality is critical**: System prompt + tool design must prevent hallucination. Responses grounded only in collected articles.
8. **No dependency on Phase 03.2**: Tool-calling queries go through Prisma, works identically on SQLite and Postgres. (Phase 03.2 is now complete anyway.)
</decisions_made>
