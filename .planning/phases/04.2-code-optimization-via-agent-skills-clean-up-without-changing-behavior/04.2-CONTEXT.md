## Implementation Decisions

### Optimization Targets
- **D-01:** Three dimensions in scope: code quality, performance patterns, and security hardening
- **D-02:** Type safety gaps: remove `any` types in API handlers (articles route where clause) and thumbnail.ts; add proper TypeScript narrowing
- **D-03:** Error handling: fix SourceList delete (no response check), FeedTable toggle rollback (optimistic update not rolled back), uncaught page-level fetch errors
- **D-04:** Security hardening: validate image URLs, enforce max pagination limit on articles API, add body validation to articles PATCH endpoint
- **D-05:** Image optimization (unoptimized flag, remotePatterns) explicitly excluded from this phase

### Skill-Driven Patterns
- **D-06:** Full audit of all components, lib modules, and API routes using react-best-practices and composition-patterns skills
- **D-07:** Not targeted to specific components; every file in features/, ui/, lib/, and app/api/ is in scope

### Verification Strategy
- **D-08:** Verification via existing test suite (briefing, chat-tools, enrich, rate-limit, sitemap, fetch-strategies) plus lint + build check
- **D-09:** No new tests required before refactoring; TypeScript compiler catches type regressions, existing tests catch logic regressions

### Work Organization
- **D-10:** Organized by layer: Plan 1 covers lib/ + API routes (type safety, error handling, security); Plan 2 covers React components (skill audit, pattern fixes)

### Claude's Discretion
- Claude decides specific pattern fixes per component based on skill audit results
- Claude decides ordering of fixes within each plan

### Folded Todos
- "Add body validation to articles PATCH endpoint" (area: api) folded into Plan 1 security hardening scope
