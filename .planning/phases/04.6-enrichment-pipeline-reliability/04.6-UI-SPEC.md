## Interaction Contracts

### Feedback Buttons (FeedbackButtons component)

**Trigger:** Up (triangle-up) or down (triangle-down) icon button on each new BriefingCard.

**Behavior:**
1. User clicks up or down arrow.
2. Button animates to active/selected state (filled color: green for up, red for down).
3. A slide-down reason panel opens immediately below the button row (no popover, no modal — inline expandable).
4. Checkboxes appear. User selects one or more reasons.
5. On checkbox selection, submit fires automatically (no explicit "Submit" button). Panel closes after 300ms.
6. `router.refresh()` is NOT called after feedback — feedback is fire-and-forget; no page reload needed.
7. If user clicks the same direction again, the selection toggles off (deselects) and closes the panel.

**Error handling:** Silent failure is intentional. If the feedback POST fails, the button state reverts to default (no selected state, no reason panel). No error toast or message is shown — feedback is non-critical and the user can retry by clicking again.

**Upvote reasons (checkboxes):**
- Directly useful for my work
- Important industry shift
- Great summary quality
- Would have missed this

**Downvote reasons (checkboxes):**
- Not relevant to me
- Too generic / low substance
- Already knew this
- Not actually important
- Duplicate coverage

**States:**
- Default: both buttons at 28x28px, `border-(--border)`, `text-(--muted-foreground)`, `variant="ghost"`
- Up hover: `border-green-500`, `text-green-500`
- Down hover: `border-red-500`, `text-red-500`
- Active/selected up: `bg-(color-mix(in srgb, #22c55e 10%, transparent))`, `border-green-500`, `text-green-500`
- Active/selected down: `bg-(color-mix(in srgb, #ef4444 10%, transparent))`, `border-red-500`, `text-red-500`

**Feedback buttons visible:** Live/catch-me-up view only. Not rendered in archive mode.

### EnrichNowButton

**Default label:** `Enrich now (N)` where N = count of pending unenriched articles.
**Loading label:** `Enriching...`
**Error label:** `Enrichment failed. Try again.` — shown inline, replaces button text for 3 seconds then resets to default label.
**Disabled:** true while loading.
**Variant:** `outline` (shadcn Button).
**On click:** calls server action `triggerEnrichment()` from `src/lib/actions.ts`, then `router.refresh()`.
**Always visible** in live view status bar, regardless of pending count (so user can always trigger on-demand).

### Pending Section (amber)

**Visibility:** Only when `pendingArticles.length > 0`.
**Position:** Below status bar, above "New since" section divider.
**Visual:** Amber left border (3px, `border-amber-600`), dimmed surface (`opacity-60` on the entire section), "Awaiting enrichment" label in 13px `text-(--muted-foreground)` italic.
**Each row:** Article title (14px, weight 400, `text-(--foreground)`) + source name + "Awaiting enrichment" label. No summary, no feedback buttons, no chat button.
**Header:** "Pending enrichment (N)" in 14px semibold `text-amber-600`. Section collapses if count is 0.

### Watermark / Reviewed Section

**Section divider — new articles:** "New since [time]" in green-500 (matches `--new-badge` value).
**Section divider — reviewed articles:** "Already reviewed this morning" / "Already reviewed" in `--muted-foreground`.
**Reviewed cards:** `opacity-50` applied to full card including topic heading. No feedback buttons. No "New" badge. Summary still visible but truncated to 1 line.

### "New" Badge

**Appearance:** 12px uppercase (`text-xs`), weight 700, letter-spacing 0.5px. Text: `text-green-500`. Background: `bg-[rgba(34,197,94,0.1)]`. Padding: 2px 6px. Border-radius: 3px.
**Note:** This uses the existing 12px badge exception (`text-xs`) already established in the design system. No new font sizes are introduced.
**Position:** Inline after article title, flex-shrink-0.
**Visibility:** Live view only, on articles enriched after watermark.

### "All Caught Up" Empty State

**Trigger:** Zero new articles since watermark.
**Layout:** Centered, `py-12` top/bottom padding.
**Icon:** CheckCircle (lucide), 32px, `text-(--muted-foreground)`.
**Heading:** "You're all caught up" — 16px, weight 600, `text-(--foreground)`.
**Body:** "No new articles since your last visit at [time]." + "Next enrichment runs at [calculated time]." — 14px, `text-(--muted-foreground)`.
**Below:** Today's reviewed articles section still renders collapsed (reference).

### Archive Banner

**Trigger:** DateStepper navigates to a past date (date param present in URL).
**Appearance:** `bg-[rgba(37,99,235,0.06)]`, `border border-[rgba(37,99,235,0.15)]`, border-radius `--radius`, 8px vertical / 16px horizontal padding (`py-2 px-4`).
**Text:** "Archive: [date]" + "Showing top 15 articles by importance. Feedback and enrichment are not available for past dates." — 13px, `text-(--foreground-secondary)`.
**Icon:** Archive (lucide), 16px.
**Position:** Between DateStepper and first topic group, margin-bottom 24px.

### Status Bar

**Layout:** Flex, space-between, full width. Background: `--muted` surface. Border: `1px solid var(--border)`. Border-radius: `--radius`. Padding: `py-2 px-4`. Margin-bottom: `mb-6`.
**Left content:** "[N] new articles since last visit ([time])" | "Last enriched: [time]"
**Right content:** EnrichNowButton (always visible)
**New count color:** `text-green-500` (green-500 = `--new-badge`)
**Zero new count color:** `text-(--foreground-secondary)` (no green when nothing is new)

## Accessibility Contract

Per CLAUDE.md WCAG 2.2 AA requirement:

- FeedbackButtons: `aria-label="Upvote: more like this"` and `aria-label="Downvote: less like this"` on each button. `aria-pressed` reflects active state.
- Reason checkboxes: wrapped in `<fieldset>` + `<legend>` ("Why did you upvote this?" / "Why did you downvote this?"). Each `<input type="checkbox">` has an explicit `<label>`.
- CaughtUpState CheckCircle icon: `aria-hidden="true"`.
- ArchiveBanner Archive icon: `aria-hidden="true"`.
- "New" badge: rendered as `<span aria-label="New article">` so screen readers announce it once per card.
- Dimmed reviewed cards: no `aria-hidden` — content is still readable by screen readers. Visual dimming via CSS `opacity-50` only.
- All interactive elements use `focus-visible:` ring, never bare `focus:`.
- Non-text contrast: feedback button borders must meet 3:1 against background. Use `--border` (#e4e4e7 on white = 1.8:1 — acceptable for decorative border; the icon itself meets 3:1).
