    **Per D-15: The frontend-design skill MUST audit the entire app for UX/UI enhancement opportunities during implementation. Not just the feed.**

    Perform a holistic design review across ALL views and components. Apply the following frontend-design audit checklist to every view:

    **1. Visual consistency audit (all views):**
    - Verify 60/30/10 color rule is maintained across feed, briefing, sources, chat, and nav
    - Check that `--primary` is reserved for: default Button, active filter tabs, search focus ring, active nav link
    - Verify no view uses `--primary` for hover states, badges, or informational highlights
    - Confirm all cards/containers use consistent border radius (`--radius` token)
    - Confirm all interactive elements use consistent hover states (`--accent`/`--muted`)

    **2. Spacing and rhythm audit:**
    - Verify consistent spacing between sections (use `gap-*` or `space-y-*`, not ad-hoc margins)
    - Check that all page containers use `section-container` (1460px) or `reading-container` (1024px) appropriately
    - Verify card internal padding is consistent across briefing cards, chat source cards, feed items

    **3. Interaction pattern consistency:**
    - All buttons use the `Button` component (no raw `<button>` elements)
    - All focus states use `focus-visible:` (not bare `focus:`)
    - All links have hover states with `transition-colors`
    - All interactive elements have appropriate cursor styles

    **4. ADHD-friendly design check:**
    - Information density: no dense text blocks; content should be scannable
    - Visual anchors: every list/grid item has an icon, avatar, or badge as a visual hook
    - Clear hierarchy: primary content (titles) visually distinct from secondary (metadata)
    - Whitespace: adequate padding and margins to prevent visual overwhelm

    **5. Cross-view consistency check:**
    - Feed view: verify fallback visuals, NEW badges, mobile list are consistent with design system
    - Briefing view: verify cards, topic groups, status indicators follow same patterns
    - Sources view: verify SourceAvatar integration, inline delete, table styling matches feed table
    - Chat view: verify message bubbles, source cards, prompt chips use design tokens
    - Nav: verify active states, hamburger menu styling, transition consistency

    **Fix any issues found.** Common issues to look for:
    - Inconsistent padding between views (one view uses `p-4`, another `p-6` for same element type)
    - Inconsistent text truncation (some titles clamp, others overflow)
    - Missing `min-w-0` on flex children (causes overflow)
    - Inconsistent empty states (different patterns for "no data" across views)
    - Inconsistent loading states (different skeleton patterns)

    **Document all findings and fixes in the SUMMARY**, including:
    - Issues found per view
    - Fixes applied
    - Items that pass without changes
    - Any additional UX improvements surfaced by the audit (per D-15 Claude's Discretion)
