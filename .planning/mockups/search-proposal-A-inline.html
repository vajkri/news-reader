Proposal A: Inline Search

  • Placement: Search input sits inline with existing filters (tabs, source, category). All controls live on one row.
  • Keyboard shortcut: Press / to focus search instantly. The / hint badge disappears when focused.
  • Debounced: Filters as you type with a 300ms debounce. No submit button needed.
  • Active search indicator: Blue tag appears in status bar showing the active search term with a clear button.
  • Highlight matching: Matching text is highlighted in yellow in article titles.
  • Combines with filters: Search works alongside source/category/read filters. All are additive (AND logic).
  • ADHD-friendly: No mode switches. Search is always visible and reachable. Clearing is one click or Escape.
  • Trade-off: Toolbar can wrap on narrow screens. On mobile, search moves to its own row naturally via flex-wrap.
