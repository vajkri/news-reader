    Per D-08, D-09, D-10, D-12, Research Pattern 5:

    Create `scripts/migrate-data.ts` that:

    1. **Reads SQLite** via `better-sqlite3` (still installed at this point):
       ```typescript
       import Database from 'better-sqlite3';
       import { PrismaClient } from '@prisma/client';

       const CUTOFF = new Date('2026-03-18T00:00:00.000Z');
       const sqliteDb = new Database('./prisma/dev.db', { readonly: true });
       const prisma = new PrismaClient(); // uses DATABASE_URL from .env.local (Neon)
       ```

    2. **Migrates all sources** (idempotent by `url` per D-10):
       - Query: `SELECT * FROM Source`
       - For each source: `prisma.source.upsert({ where: { url }, create: { name, url, category }, update: {} })`
       - Log each source name

    3. **Migrates articles from March 18th 2026 onward** (per D-09, idempotent by `guid` per D-10):
       - Query: `SELECT * FROM Article WHERE publishedAt >= '2026-03-18T00:00:00.000Z' ORDER BY publishedAt ASC`
       - For each article:
         a. Check if `guid` already exists in Postgres (skip if so)
         b. Look up source in SQLite by `sourceId` to get source `url`
         c. Find corresponding source in Postgres by `url` (NOT by id, since IDs may differ)
         d. Create article with all fields, converting:
            - `isRead`: SQLite stores as integer 0/1, convert with `Boolean(a.isRead)`
            - `topics`: SQLite stores as JSON string, parse with `a.topics ? JSON.parse(a.topics) : null`
            - `publishedAt`: Convert string to Date with `new Date(a.publishedAt)`
            - `enrichedAt`: Convert string to Date with `new Date(a.enrichedAt)`
       - Count created vs skipped, log results

    4. **Cleanup**: Close SQLite connection and Prisma client in `finally` block.

    5. **Run the script**:
       ```bash
       npx tsx scripts/migrate-data.ts
       ```

    6. **Commit the script** per D-11 (will be deleted after verification, preserved in git history).
