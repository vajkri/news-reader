## What Was Built

**POST /api/feedback** (`src/app/api/feedback/route.ts`):
- Validates articleId, direction (up/down), and reasons against VALID_UP_REASONS (4 items) / VALID_DOWN_REASONS (5 items) allow-lists
- Denormalizes sourceId, topics, contentType from article at feedback time (per D-18)
- Returns 400 for invalid input, 404 for missing article, 200 on success
- No auth required (single-user app), append-only writes

**FeedbackButtons** (`src/components/features/briefing/FeedbackButtons.tsx`):
- `'use client'` component with ChevronUp/ChevronDown Lucide icons
- Uses `<Button variant="ghost" size="vote">` (28px h-7 w-7) per CLAUDE.md no-raw-button rule
- Inline reason checkbox panel using `<fieldset>` + `<legend>` for WCAG 2.2 AA
- `aria-label="Upvote: more like this"` / `aria-label="Downvote: less like this"` + `aria-pressed`
- Auto-submits POST /api/feedback on checkbox selection, closes panel after 300ms
- Silent failure: state reverts to default on fetch error, no error UI shown
- Green active state for up (color-mix #22c55e 10%), red for down (#ef4444 10%)

**Button component** (`src/components/ui/Button.tsx`):
- Added `"vote"` to size union: `"sm" | "default" | "lg" | "icon" | "vote"`
- Size mapping: `"h-7 w-7 p-0": size === "vote"` (28px touch target)

**BriefingCard** (`src/components/features/briefing/BriefingCard.tsx`):
- Added `isNew?: boolean` and `isArchive?: boolean` props
- Renders `<FeedbackButtons articleId={article.id} sourceId={article.sourceId} />` only when `!isArchive`
- Bottom bar uses `flex items-center justify-between` with Chat button left, FeedbackButtons right
- `isNew` prop renders "New" badge inline after article title

## Decisions Made

- **Button size="vote"** over raw button: CLAUDE.md mandates using Button component; vote size variant added to satisfy 28px UI-SPEC requirement
- **sourceId kept in FeedbackButtons props**: Interface-stable for future use even though API fetches denormalized value server-side
- **No router.refresh()**: Feedback is fire-and-forget per UI-SPEC; no page reload needed after submission
