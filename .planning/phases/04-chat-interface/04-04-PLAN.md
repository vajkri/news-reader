    2. In `src/lib/chat-tools.ts`, replace the `searchArticlesTool.execute` implementation:

    Replace the Prisma `findMany` with `contains` with a `$queryRaw` query using Postgres full-text search:

    ```typescript
    execute: async ({ query, limit }) => {
      const articles = await prisma.$queryRaw<Array<{
        id: number;
        title: string;
        summary: string | null;
        link: string;
        publishedAt: Date | null;
        importanceScore: number | null;
        topics: unknown;
        sourceName: string;
        sourceCategory: string | null;
      }>>`
        SELECT
          a.id, a.title, a.summary, a.link,
          a."publishedAt", a."importanceScore", a.topics,
          s.name AS "sourceName", s.category AS "sourceCategory"
        FROM "Article" a
        JOIN "Source" s ON a."sourceId" = s.id
        WHERE a."enrichedAt" IS NOT NULL
          AND (
            to_tsvector('english', coalesce(a.title, '') || ' ' || coalesce(a.description, '') || ' ' || coalesce(a.summary, ''))
            @@ plainto_tsquery('english', ${query})
          )
        ORDER BY
          ts_rank(
            to_tsvector('english', coalesce(a.title, '') || ' ' || coalesce(a.description, '') || ' ' || coalesce(a.summary, '')),
            plainto_tsquery('english', ${query})
          ) DESC,
          a."importanceScore" DESC NULLS LAST,
          a."publishedAt" DESC NULLS LAST
        LIMIT ${limit}
      `;

      return articles.map(a => ({
        id: a.id,
        title: a.title,
        summary: a.summary,
        source: a.sourceName,
        publishedAt: a.publishedAt,
        link: a.link,
        importanceScore: a.importanceScore,
        topics: (a.topics as string[] | null) ?? [],
      }));
    },
    ```

    Key details:
    - Use Prisma's tagged template literal `$queryRaw` (NOT `$queryRawUnsafe`) for automatic parameterization and SQL injection safety
    - `plainto_tsquery` handles multi-word queries naturally (splits on spaces, treats as AND)
    - Search across title + description + summary for maximum recall
    - `ts_rank` orders by relevance, then importance, then recency
    - `coalesce` handles nullable fields safely
