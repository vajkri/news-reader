## Spacing Scale

Declared values (multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-label gaps (`gap-1`), inline badge padding |
| sm | 8px | Button internal padding, row cell gaps (`gap-2`) |
| md | 16px | Card padding, toolbar spacing, section inner padding |
| lg | 24px | Section padding (`--container-padding: 1.5rem`), card group separation |
| xl | 32px | Layout gaps between major page sections |
| 2xl | 48px | Vertical rhythm between topic groups on briefing |
| 3xl | 64px | Page-level top/bottom breathing room |

Exceptions:
- Touch targets on mobile: minimum 44px height for all interactive elements (buttons, row actions, nav links)
- Header height: 48px (`h-12`) — fixed, do not change
- Feed row thumbnail: 40x40px (`h-10 w-10`)
- Feed fallback avatar (source initial): 32x32px (`h-8 w-8`) — compact row density

## Color

All values from `src/app/globals.css`. Phase 5 enforces correct token usage; no new tokens introduced.

| Role | Light value | Dark value | Usage |
|------|-------------|------------|-------|
| Dominant (60%) | `#ffffff` | `#09090b` | Page background, card backgrounds (`--background`, `--card`) |
| Secondary (30%) | `#f4f4f5` | `#27272a` | Toolbar surfaces, hover states, muted backgrounds (`--muted`, `--secondary`, `--accent`) |
| Accent (10%) | `#18181b` | `#fafafa` | Reserved elements only (see below) (`--primary`) |
| Destructive | `#dc2626` | `#f87171` | Delete source button only (`--destructive`) |

Accent (`--primary`) reserved for:
1. Default-variant Button (filled CTA)
2. Active nav link indicator (current page)
3. Search input focus ring
4. Active filter tab / selected chip state

`--primary` is NOT used for: hover states (use `--accent`/`--muted`), badges, informational highlights, NEW badges, or source avatars.

NEW badge color: `Badge` component with `secondary` variant (zinc muted surface) + small dot or label. Consistent with briefing "New" treatment.

Source initial avatar palette (Claude's discretion — 6 distinct hues, accessible on dark circle backgrounds):
- Use `bg-blue-600`, `bg-violet-600`, `bg-emerald-600`, `bg-amber-600`, `bg-rose-600`, `bg-sky-600`
- Text on all: `text-white` (sufficient contrast on all 600-level backgrounds)
- Applied deterministically: hash source name to index mod 6

Dark mode audit is required (D-11): verify `--card: #0f0f12` elevation, `--border: #27272a` visibility, and all text tokens meet WCAG 2.2 AA (4.5:1 body, 3:1 large text).

## Visual Hierarchy

### Feed desktop focal point

Primary visual anchor on feed desktop: article title cell (`text-sm font-medium`, `text-(--foreground)`). `SourceAvatar` / `TopicIcon` serves as secondary pull — the colored circle or icon draws the eye to each row's left edge, then the title carries reading focus across the row. Metadata cells (source name, published time, badges) are tertiary and use `text-(--muted-foreground)` to recede.

### Card primitive decision

Use independently styled components (no shared `Card` primitive). Rationale: briefing cards, feed rows, and source list items have sufficiently different layouts that a shared Card wrapper would require too many overrides. Apply consistent token usage (`--card`, `--border`, `--radius`) directly in each component.

## Interaction Contracts

### Active nav link
- Detect active route via `usePathname()` in a `'use client'` nav wrapper component
- Active style: `text-(--foreground) font-semibold` (weight change + full opacity)
- Inactive style: `text-(--muted-foreground) hover:text-(--foreground) transition-colors` (existing)
- No underline, no background highlight — weight + color is sufficient

### Hamburger menu (mobile, below 640px)
- Trigger: `Button` with `variant="ghost" size="icon"`, icon `Menu` (Lucide), visible only on `<sm`
- Drawer: full-height side sheet from left, overlays content, backdrop scrim `bg-black/40`
- Nav links inside drawer: same 3 links (Feed, Briefing, Sources), `text-base font-medium`, `p-4` touch targets
- Close: tap outside scrim or tap X button inside drawer
- Keyboard: `Escape` closes drawer; focus trap inside drawer while open

### Feed NEW badge
- Source: articles fetched since the last feed page visit (watermark key: `feed_watermark`, separate from `briefing_watermark`)
- Badge: `Badge` component, `secondary` variant, label "New", `text-xs`
- Position: inline after article title, same line
- Watermark updated: on feed page mount (client-side), sets watermark to `Date.now()`
- Articles with NEW badge: unread only (read articles never show NEW)

### Feed fallback visuals
- Enriched article (has topics): `TopicIcon` — Lucide icon mapped from first topic string, 16px, `text-(--muted-foreground)`, centered in 40x40 `bg-(--muted) rounded` container
- Unenriched article (no topics): `SourceAvatar` — first letter of source name, colored circle per palette above
- Article with real thumbnail: existing `Image` component (unchanged)

### Feed mobile layout (below 640px)
- Table view hidden: `hidden sm:block` on table wrapper
- Mobile list shown: `block sm:hidden` stacked list
- Each row: flex column. Top row: `SourceAvatar`/`TopicIcon` (32px) + title (line-clamp-2, `text-sm font-medium`). Bottom row: source name + published time + action buttons
- Row padding: `py-3 px-4`, `border-b border-(--border)`
- Actions row: read toggle + chat button, `text-xs`, full touch targets (min 44px height)

### Chat panel mobile
- Below 640px: panel takes full viewport width (`w-dvw`), positioned fixed, no resize handle
- Above 640px: existing desktop embedded/overlay behavior unchanged

### Focus styles
- All interactive elements: `focus-visible:outline-2 focus-visible:outline-offset-2 focus-visible:outline-(--primary)`
- Never use bare `focus:` — `focus-visible:` only

## Briefing UX States Contract

All 5 states must be visually correct and mobile-responsive (D-16 directive):

| State | Trigger | Required elements |
|-------|---------|-------------------|
| Morning visit (new articles) | Articles since watermark exist | New section with NEW badges, reviewed section dimmed |
| Caught up | No articles since watermark | CaughtUpState component, positive reinforcement copy |
| Archive | Date param present + not today | ArchiveBanner, top 15 by importance, no feedback/enrich controls |
| Pending enrichment | Unenriched articles exist | PendingSection with amber styling, "Awaiting enrichment" labels |
| Evening visit (mixed) | Mix of new + reviewed | Both sections visible, SectionDivider between them |
