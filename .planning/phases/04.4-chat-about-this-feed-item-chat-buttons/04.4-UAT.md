### 1. Actions column in feed table
expected: The feed table's last column is an Actions column (no header text). Each row shows two icon buttons side by side: a circle icon for read/unread toggle (blue circle = unread, check-circle = read), and a message-circle icon for "Chat about this". The old text-based "Status" column is gone.
result: pass

### 2. Read/unread toggle works
expected: Clicking the circle icon toggles the article between read and unread states. The icon changes between an empty circle (unread, blue) and a checked circle (read, muted). The article title dims when marked as read.
result: issue
reported: "The mark as (un)read button is buggy, it doesn't toggle right"
severity: major
root_cause: "Neon Postgres HTTP driver does not support transactions. Prisma update with include generates implicit transaction. Removed include from PATCH route."
fix: Removed `include: { source }` from /api/articles/[id] PATCH route
retest: pass (Option C labels implemented, min-width on toggle)

### 3. Chat about this from feed
expected: Clicking the message-circle icon on a feed row opens the chat panel. The panel shows a pinned context card at the top with that article's title, source name, and date. Contextual prompt chips appear below. Input placeholder says "Ask anything about this article..."
result: pass

### 4. "Read this for me" chip with Sparkles icon
expected: Among the contextual prompt chips, "Read this for me" appears with a small sparkles icon to the left of the text. The old "Summarize this article" chip is gone. Other chips remain: "Why does this matter?", "Related articles", "Compare to competitors".
result: pass

### 5. "Read this for me" produces detailed summary
expected: Clicking the "Read this for me" chip triggers the AI to fetch the full article content and generate a detailed, scannable summary. The response should be a 1-2 minute read with a bold lead sentence, short paragraphs, bullet points where appropriate, and a "Why it matters" closing. Noticeably more detailed than the 2-3 sentence enrichment summary.
result: issue
reported: "Initial: AI said 'no articles found' (missing article URL in context). After fix: fetchable sources work but unfetchable sources (Medium, OpenAI) got 403 with no fallback."
severity: major
root_cause: "1) Article link not passed in context so AI couldn't fetch directly. 2) Bot UA blocked by many sites. 3) No paste fallback for unfetchable sources."
fix: Added link + fetchable flag to article context. Updated UA. Unfetchable sources skip fetch and immediately ask user to paste article text.
retest: pass

### 6. BriefingCard chips also updated
expected: On the /briefing page, clicking "Chat about this" on a BriefingCard also shows "Read this for me" with Sparkles icon instead of the old "Summarize this article" chip. Same chip set as the feed.
