## Decisions Made

- **Neon Marketplace provisioning**: Vercel auto-injects DATABASE_URL (pooled PgBouncer) and DATABASE_URL_UNPOOLED (direct). Both env vars pulled locally via `vercel env pull .env.local`.
- **directUrl for migrations**: PgBouncer pooler doesn't support transaction-level DDL. `directUrl = env(DATABASE_URL_UNPOOLED)` lets `prisma migrate dev` use the direct connection while the app uses the pooled URL.
- **Json? instead of String?**: Postgres JSONB is a first-class type. Storing topics as `Json?` removes all stringify/parse overhead and makes queries on topic data possible in future.
- **Deleted SQLite migrations**: The two old migrations (20260318072441, 20260319183507) are incompatible with Postgres DDL. Replaced with a single fresh `20260321133925_init` migration.
- **postinstall script**: `"postinstall": "prisma generate"` added after `"test"` in package.json. Ensures Prisma Client is regenerated after npm install on Vercel build workers (avoids stale cached client mismatch).
