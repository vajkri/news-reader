# Phase 04.4: Chat about this: feed item chat buttons - Context

**Gathered:** 2026-03-28
**Status:** Ready for planning

<domain>
## Phase Boundary

Add "Chat about this" buttons to feed article rows and introduce a "Read this for me" AI shortcut that fetches full article content and generates a detailed, ADHD-friendly summary (1-2 min read). This replaces the need to visit noisy original sites. The existing `chat-about-this` event pattern, ChatWrapper, and ChatPanel infrastructure from Phase 4 are reused. No new pages or chat API changes beyond adding a URL-fetch tool.

</domain>

<decisions>
## Implementation Decisions

### Button placement
- **D-01:** New "Actions" column as the last column in the feed table, consolidating Read/Unread toggle + "Chat about this" button. Extensible for future actions.
- **D-02:** The existing standalone "Status" (isRead) column is removed; its toggle moves into the Actions column.

### Context payload
- **D-03:** Reuse the existing ArticleContext type (id, title, source name, publishedAt). Same data as BriefingCard passes. No extended enrichment data needed.

### Prompt chips
- **D-04:** Replace "Summarize this article" with "Read this for me" in the CONTEXTUAL_CHIPS array. This applies to both BriefingCard and feed item contexts (consistent UX everywhere).
- **D-05:** The "Read this for me" chip gets a Sparkles icon (from Lucide) to visually mark it as an AI-enhanced shortcut, distinguishing it from regular conversation starters.
- **D-06:** Other contextual chips remain unchanged: "Why does this matter?", "Related articles", "Compare to competitors".

### "Read this for me" feature
- **D-07:** On-demand content fetching. When the user triggers "Read this for me", a chat tool fetches the article URL, extracts the readable text content, and generates a detailed ADHD-friendly summary.
- **D-08:** Target output length: 1-2 minute read. More detailed than the existing 2-3 sentence enrichment summary, but structured for scannability (short paragraphs, key points, no fluff).
- **D-09:** The purpose is to replace visiting the original article site, avoiding ad-heavy, distraction-laden pages (e.g., TechCrunch). The summary should capture all important information from the article.

### Claude's Discretion
- Actions column layout (icon-only buttons vs text labels, spacing)
- URL content extraction approach (readability library, fetch strategy)
- "Read this for me" system prompt wording for ADHD-friendly output format
- Error states (article behind paywall, fetch failure, content too short to summarize)
- Actions column responsive behavior on smaller screens

</decisions>

<specifics>
## Specific Ideas

- The "Read this for me" feature is specifically motivated by avoiding distraction-heavy news sites (TechCrunch, etc.) with blinking ads. The summary replaces the need to visit the original site entirely.
- Sparkles icon on the chip should be subtle, not flashy. It's a functional marker, not decoration.
- Actions column should feel compact. Icon buttons preferred over text-heavy buttons in a table context.
