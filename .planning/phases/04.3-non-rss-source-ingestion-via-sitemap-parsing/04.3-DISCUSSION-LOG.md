## Content Extraction

### Q1: How should we extract article content from sitemap-discovered URLs?

| Option | Description | Selected |
|--------|-------------|----------|
| HTML scrape + readability (Recommended) | Fetch each page, use readability library to extract title, description, content, thumbnail. Same quality as RSS. | Y |
| Meta tags only | Fetch page, extract only title, meta description, og:image. Lighter but less content for enrichment. | |
| URL + date only | Store just URL and lastmod. Let enrichment fetch pages itself. | |

**User's choice:** HTML scrape + readability
**Notes:** None

### Q2: Should readability-extracted content replace the RSS description field or be stored separately?

| Option | Description | Selected |
|--------|-------------|----------|
| Same description field (Recommended) | Store in existing Article.description. Enrichment works identically for all source types. | Y |
| New fullContent field | Add separate column for scraped content. More explicit but adds schema complexity. | |

**User's choice:** Same description field
**Notes:** None

### Q3: How should we handle thumbnail extraction for sitemap sources?

| Option | Description | Selected |
|--------|-------------|----------|
| og:image from meta tags (Recommended) | Extract og:image or twitter:image from HTML. Falls back to null. | Y |
| First image in content | Use readability to find first img in article body. May grab decorative images. | |
| Skip thumbnails | Don't extract for sitemap sources. UI handles null thumbnails. | |

## Sitemap Filtering

### Q1: How should we filter sitemap URLs to only ingest actual articles?

| Option | Description | Selected |
|--------|-------------|----------|
| Path pattern per source (Recommended) | Each source defines a URL path pattern (e.g., /news/*). Simple, explicit. | Y |
| Dedicated article sitemap URL | Point at sub-sitemap if site provides one. Relies on site structure. | |
| Heuristic filtering | Auto-detect article URLs by path patterns. Flexible but less predictable. | |

**User's choice:** Path pattern per source
**Notes:** None
