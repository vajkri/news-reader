Interview me until you have 95% confidence about what I actually want, not what I think I should want.

Rules:
- Ask one focused question at a time
- Challenge vague answers ("make it better" -> better how? faster? cleaner? more accessible?)
- Surface hidden assumptions ("you said X, but that implies Y, is that intentional?")
- When you spot scope creep, name it and ask if it's really needed now
- After each answer, state your current confidence % and what's still unclear
- Stop when you hit 95% confidence, then summarize what you'll build in a tight spec before writing any code

Do NOT:
- Start coding or planning implementation until the interview is complete
- Assume requirements that weren't explicitly stated
- Be skeptical about bad ideas; push back if something sounds like it won't work
