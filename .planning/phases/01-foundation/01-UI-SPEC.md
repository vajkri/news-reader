## Spacing Scale

8-point scale. All values are multiples of 4, plus one deliberate in-between value.

| Token | Value | Tailwind | Usage |
|-------|-------|----------|-------|
| xs | 4px | `gap-1`, `p-1` | Icon gaps, tight inline padding |
| sm | 8px | `gap-2`, `p-2` | Compact controls, badge padding |
| sm+ | 12px | `py-3` | FeedToolbar top/bottom padding (deliberate in-between value) |
| md | 16px | `gap-4`, `p-4` | Default element spacing, toolbar gaps |
| lg | 24px | `gap-6`, `p-6` | Section padding, card inner padding |
| xl | 32px | `gap-8`, `p-8` | Layout column gaps |
| 2xl | 48px | `gap-12`, `py-12` | Major section breaks |
| 3xl | 64px | `gap-16`, `py-16` | Page-level vertical spacing |

Exceptions:
- Search input minimum touch target: 36px height (matches existing Input `h-9`) — no exception needed, within scale
- Toolbar height: 48px (`h-12`) — next multiple of 4 above 44px minimum touch target
- Sticky header height: 48px (`h-12`) — matches mockup `height: 3rem`

## Typography

Two fonts. Four roles. Two weights.

| Role | Font | Size | Weight | Line Height | Usage |
|------|------|------|--------|-------------|-------|
| Body | Geist Sans | 14px | 400 | 1.5 | Article metadata, filter labels, secondary text |
| UI Label | Geist Sans | 14px | 600 | 1.4 | Button labels, nav links, badge text, status bar |
| Heading | Geist Sans | 16px | 600 | 1.4 | Column headers, section titles, toolbar groupings |
| Mono | Geist Mono | 13px | 400 | 1.5 | Code references, URL display (if any) |

Notes:
- Exactly two weights in use: 400 (regular) and 600 (semibold).
- Article title in feed rows: 14px, weight 400, line-height 1.5 — same as Body. Bold variant (weight 600) for unread articles is left to Claude's discretion.
- Minimum font size: 13px (Geist Mono only). No UI text below 13px.
- Source: CONTEXT.md (Geist Sans/Mono decision); sizes derived from existing component code (`text-sm` = 14px, `text-xs` = 12px in badge — badge is the sole 12px exception documented below).

Badge exception: Badge component uses `text-xs` (12px). This is the only element below 13px and is acceptable for short categorical labels with adequate contrast. Do not add new 12px elements.

## Color

Dual-mode (light/dark via `prefers-color-scheme`). All values from `globals.css`.

### Light Mode

| Role | Variable | Value | Usage |
|------|----------|-------|-------|
| Dominant (60%) | `--background` | `#ffffff` | Page background, card backgrounds |
| Secondary (30%) | `--muted` / `--secondary` | `#f4f4f5` | Toolbar background, hover states, muted surfaces |
| Accent (10%) | `--primary` | `#18181b` | Primary CTA buttons, active nav link, active filter tab, search focus ring |
| Destructive | `#ef4444` (red-500) | `bg-red-500` | Destructive Button variant only |
| Border | `--border` | `#e4e4e7` | All dividers, input borders, card borders |
| Muted text | `--muted-foreground` | `#71717a` | Placeholder text, secondary metadata, inactive nav |

### Dark Mode

| Role | Variable | Value | Usage |
|------|----------|-------|-------|
| Dominant (60%) | `--background` | `#09090b` | Page background |
| Secondary (30%) | `--muted` / `--secondary` | `#27272a` | Toolbar, hover, muted surfaces |
| Accent (10%) | `--primary` | `#fafafa` | Same reserved elements as light mode |
| Destructive | `#ef4444` | unchanged | Destructive actions only |
| Border | `--border` | `#27272a` | Dividers, input borders |

Accent (`--primary`) is reserved for:
1. Default Button variant (primary CTA)
2. Active/selected state on filter tabs (read filter: All / Unread / Read)
3. Search input focus ring (`focus-visible:ring-[var(--primary)]`)
4. Active nav link highlight

Accent is NOT used for: hover states (use `--accent`/`--muted`), badges, informational highlights.

Search match highlight: `bg-yellow-200 dark:bg-yellow-800` — outside the 60/30/10 token system, applied only to `<mark>` spans inside article title text. This is a functional highlight, not a brand accent.

## Interaction Contracts

### Search Input (FOUND-02)

- Placement: inline in FeedToolbar, left side, alongside source/category/read filters
- Width: `w-48` collapsed, expands to `w-64` on focus (or fixed `w-56` if layout permits)
- Placeholder: "Search articles..." (see Copywriting section)
- Debounce: 300ms between last keypress and API call
- Keyboard: `/` focuses input (when no input is active), `Esc` clears value and blurs
- Active indicator: when `debouncedSearch` is non-empty, status bar shows "X results for '[query]'" with a clear button (ghost Button, `size="sm"`)
- Match highlight: `<mark className="bg-yellow-200 dark:bg-yellow-800 rounded-sm px-0.5">` wrapping matched substring in article title column only
- No submit button

### Infinite Scroll (FOUND-01)

- Sentinel element: `<div ref={sentinelRef} />` placed immediately below last rendered table row
- Loading indicator: spinner or skeleton rows — Claude's discretion on visual treatment (see below)
- Status bar: "Showing {n} of {total} articles" — updates after each page loads
- End state: when `hasMore === false`, status bar shows "Showing all {total} articles"
- No "load more" button; no page number UI

### Loading Skeleton (Claude's Discretion)

Use 3 skeleton rows during the first page load and between page appends. Each skeleton row mirrors the table row height (`h-10`). Skeleton rows: `bg-[var(--muted)] animate-pulse rounded-sm`. This is the recommended treatment — executor may adjust if the table structure makes row-height skeletons impractical.

### Filter Tabs (Read Filter)

The existing raw `<button>` filter tabs (All / Unread / Read) must be migrated to `Button` component with `variant="ghost"` for inactive and `variant="outline"` for active. Active tab also gets `font-semibold` text weight.

### Focus Management

All focusable elements use `focus-visible:` (not `focus:`) for ring styles. The Input component bug (uses bare `focus:`) is fixed during Phase 1.

## Copywriting Contract

| Element | Copy | Notes |
|---------|------|-------|
| Search placeholder | "Search articles..." | Input placeholder in FeedToolbar |
| Search active indicator | "Showing {n} results for \"{query}\"" | Status bar when search active |
| Search clear button label (aria) | "Clear search" | `aria-label` on ghost Button |
| Empty state (no results) | Heading: "No articles found" / Body: "Try a different keyword or clear the search." | Shown when search returns 0 results |
| Empty state (no articles at all) | Heading: "No articles yet" / Body: "Add a source to start collecting articles." | Shown before any RSS fetch has run |
| Loading more indicator (aria) | "Loading more articles" | `aria-label` on spinner/skeleton region |
| Status bar (loading) | "Loading..." | Brief text while first page loads |
| Status bar (loaded) | "Showing {loaded} of {total} articles" | Updates per page |
| Status bar (end of feed) | "Showing all {total} articles" | When hasMore is false |
| Cron auth error (401 response body) | "Unauthorized" | Server response text -- no UI, HTTP only |
| Keyboard shortcut hint (optional) | "Press / to search" | Tooltip or ghost text below search; only if space allows |

No destructive actions in Phase 1. No confirmation dialogs needed.

No em dashes in any copy. Use commas or periods instead.

## Layout Contract

### Primary Visual Anchor

The article title column in FeedTable commands first attention: it is the widest column, rendered at body size with weight 600 on unread items, and is the direct target of search highlight styling.

### Page Structure

```
<header>              sticky, h-12, border-bottom, backdrop-blur
  .section-container
    logo | nav (Feed, Sources)
</header>

<main>
  .section-container  (max-w-[1800px] mx-auto px-6)
    <FeedToolbar />   search input + filters
    <FeedTable />     article rows + sentinel div
    <StatusBar />     "Showing X of Y articles"
</main>
```

### Container Utility

The `.section-container` utility must be added to `globals.css` before other Phase 1 work lands:

```css
:root {
  --container-width: 100dvw;
  --container-max-width: 1800px;
  --container-padding: 1.5rem;
}

@layer components {
  .section-container {
    width: var(--container-width);
    max-width: var(--container-max-width);
    margin-inline: auto;
    padding-inline: var(--container-padding);
  }
}
```

Replace all `max-w-6xl mx-auto px-4` usages with `.section-container`.

### FeedToolbar Layout

Horizontal row. Left side: search input. Right side: source select, category select, read filter tabs. Gap between groups: `gap-2` (8px). Toolbar top/bottom padding: `py-3` (12px — deliberate in-between value, see Spacing Scale).

## Accessibility Contract

- All interactive elements reachable via keyboard Tab order
- Search input: `aria-label="Search articles"` (in addition to visible placeholder)
- Infinite scroll sentinel: `aria-hidden="true"` (it is presentational only)
- Skeleton loading region: `aria-busy="true"` on the table container during load
- Status bar: `aria-live="polite"` so screen readers announce count updates
- Match highlight `<mark>` elements: no additional aria needed (semantic `<mark>` is sufficient)
- Filter tabs: `aria-pressed` on active tab button
- Keyboard shortcut `/`: guard with `document.activeElement?.tagName !== "INPUT"` to prevent triggering inside other inputs
