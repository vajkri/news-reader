## What Was Built

### Watermark Library (src/lib/watermark.ts)

- `getWatermark()`: reads `briefing_watermark` from `UserPreference` DB; seeds to 24h ago on first visit
- `updateWatermark(ts)`: upserts ISO string to `UserPreference` table
- Uses `import 'server-only'` and same upsert pattern as existing `RateLimit` model
- 3 unit tests covering stored watermark, first-visit default, and upsert call shape

### New UI Components

- **SectionDivider**: labeled horizontal rule with `variant="new"` (green-500) or `variant="reviewed"` (muted-foreground)
- **CaughtUpState**: centered empty state with CheckCircle icon (32px, aria-hidden), "You're all caught up" heading, last visit time
- **ArchiveBanner**: blue info banner (rgba(37,99,235,0.06) bg, rgba border) with Archive icon, top-15 copy, date label
- All three exported from barrel `index.ts`

### Briefing Page Rewrite (src/app/briefing/page.tsx)

**Live mode (no date param or date = today):**
1. `getWatermark()` snapshot, then `updateWatermark(new Date())` immediately
2. New articles: `enrichedAt: { not: null, gt: watermark }` — all of them, no limit
3. Reviewed articles: `enrichedAt: { not: null, lte: watermark }` + today's `publishedAt` window, top 15
4. Renders `CaughtUpState` when no new articles, otherwise `SectionDivider` + topic groups with `isNew` badge
5. Reviewed section dimmed at `opacity-50`

**Archive mode (past date param):**
1. No watermark read/write
2. `publishedAt` day boundary query, `take: 15`, `orderBy: importanceScore desc`
3. `ArchiveBanner` between DateStepper and content
4. `isArchive` prop hides "Chat about this" button on all cards

### BriefingCard / TopicGroup Changes

- `BriefingCard` gains `isNew?: boolean` (renders "New" badge span with `aria-label="New article"`) and `isArchive?: boolean` (hides chat button)
- `TopicGroup` gains `isNew?` and `isArchive?` passthrough props to each `BriefingCard`
